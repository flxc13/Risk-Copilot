4. Component Summary

Application Load Balancer

Public entry point for users. It receives HTTP traffic on port 80 and routes requests to the ECS service.

ECS Fargate Service

Runs and maintains the FastAPI Docker container without managing EC2 servers. Desired task count is set to 1 for the lab deployment.

Amazon ECR

Stores the Docker image for the application, for example risk-copilot:latest.

AWS Secrets Manager

Stores sensitive runtime values such as POE_API_KEY. ECS injects the value as an environment variable.

CloudWatch Logs

Centralizes container logs under the log group /ecs/risk-copilot.

IAM Task Execution Role

Grants ECS permission to pull images from ECR, retrieve secrets, and publish logs to CloudWatch.

5. Architecture Description for README

The application is deployed as a containerized FastAPI service on AWS ECS Fargate. User traffic enters through an internet-facing Application Load Balancer, which forwards HTTP requests to an ECS service running inside a VPC. The container image is stored in Amazon ECR, runtime secrets are injected from AWS Secrets Manager, and application logs are centralized in CloudWatch Logs. IAM roles are used to enforce controlled access for image pulls, secret retrieval, and logging.
